# Contributing to Workforce Scheduling Platform

First off, thank you for considering contributing to the Workforce Scheduling Platform! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

## üöÄ How to Contribute

### Reporting Bugs
- Always search for existing issues before creating a new one.
- Use a clear and descriptive title.
- Describe the exact steps which reproduce the problem.

### Suggesting Enhancements
- Check if the feature has already been suggested.
- Explain why this enhancement would be useful to most users.

### Pull Requests
1. Fork the repository.
2. Create your feature branch (`git checkout -b feature/AmazingFeature`).
3. Commit your changes (`git commit -m 'Add some AmazingFeature'`).
4. Push to the branch (`git push origin feature/AmazingFeature`).
5. Open a Pull Request.

## üíª Style Guides

### Git Commit Messages
- Use the present tense ("Add feature" not "Added feature").
- Use the imperative mood ("Move cursor to..." not "Moves cursor to...").
- Limit the first line to 72 characters or less.

### JavaScript/React Style Guide
- Use functional components and hooks.
- Follow ESLint configurations provided in the project.

### Python Style Guide
- Follow [PEP 8](https://www.python.org/dev/peps/pep-0008/) style guide.
- Use type hints for all function signatures.
